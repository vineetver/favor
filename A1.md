0) Core rule before the diagrams

Entity pages = reference + evidence + pivots
Analysis/Signal pages = prioritization + decisions

For now (since you want variant + gene only), we’ll still support prioritization on those pages via a right rail + “Open in Analysis” CTA, but we won’t force everything into tabs.

1) Navigation model (Variant + Gene only)
Route map (desktop)
/search  ──► /variant/:id
         └──► /gene/:id

/variant/:id
  ├─ overview      (default)       -> /variant/:id
  ├─ evidence                      -> /variant/:id/evidence
  ├─ network                       -> /variant/:id/network
  ├─ browser                       -> /variant/:id/browser
  └─ analysis (optional context)   -> /variant/:id/analysis?signal_id=...

/gene/:id
  ├─ overview      (default)       -> /gene/:id
  ├─ disease-therapeutics          -> /gene/:id/disease-therapeutics
  ├─ regulation                    -> /gene/:id/regulation
  ├─ variants                      -> /gene/:id/variants
  ├─ network                       -> /gene/:id/network
  ├─ evidence                      -> /gene/:id/evidence
  └─ analysis (optional context)   -> /gene/:id/analysis?signal_id=...
Why this works

Major “tabs” become routes

Sections within each route become scroll sections

Local tabs only used inside a section (chart/table, transcript selector, etc.)

     • Add to compare set
     • Add to report draft
     • Add to cohort watchlist

  ├─ Variant summary (PDF)
  ├─ Variant + candidate genes
  ├─ Variant + trait context
  └─ Include provenance appendix [checkbox]

[Sort: Prioritization score ▼]
  ├─ Prioritization score
  ├─ L2G score
  ├─ cCRE support score
  ├─ Distance to TSS
  ├─ VEP / coding impact

[Summary | cCRE Table | Tissue Matrix]

10) Design rules for your frontend team (pin this in Slack)

These rules are what prevent future chaos when someone wants to add “just one more tab.”

RULE 1: Major perspectives are routes, not tabs.
RULE 2: A page answers one primary question.
RULE 3: Sections are stacked + hide if empty.
RULE 4: Summary first, ranked evidence second, raw data last.
RULE 5: Provenance lives in a reusable drawer and/or evidence route.
RULE 6: Context filters are sticky and shared across routes.
RULE 7: Prioritization actions live in right rail + analysis route, not buried in content.
RULE 8: Use local tabs only for small view toggles (table/chart, summary/raw, transcript, etc.).
